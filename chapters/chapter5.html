1. Introduction to Hash Functions

What is a Hash Function?

A hash function is a mathematical algorithm that converts data of any size into a fixed-length string called a hash value (or digest). It creates a unique digital fingerprint for the input, ensuring consistency in output length regardless of input size. Hashing is a one-way function — once data is hashed, it is computationally infeasible to retrieve the original data from the hash value.

Hash functions are fundamental building blocks of modern security systems. They are used everywhere — from password storage and data integrity verification to digital signatures and blockchain technology.

Key Property: A hash function is a one-way function. It is easy to compute the hash from the input, but computationally infeasible to find the input from the hash. This is what makes hashing useful for security.

2. How Hash Functions Work

Internal Process

When data is hashed, a complex process occurs behind the scenes. The input is divided into fixed-size blocks, processed through multiple rounds of mathematical transformations, and produces a unique fixed-size output.

Step-by-Step Process

  1. Input Processing: The input data (message) is divided into fixed-size blocks (e.g., SHA-256 uses 512-bit blocks).
  2. Padding: If the final block isn't large enough, the algorithm adds extra bits (padding) to ensure consistent processing.
  3. Compression: Each block passes through a compression function that applies a series of mathematical transformations.
  4. Output: The final result is a unique, fixed-size hash value (message digest). The length depends on the algorithm used.

Avalanche Effect Example: Even changing a single character (e.g., "Hello" vs "hello") produces a completely different hash output. This is called the avalanche effect and is critical for security.

3. Properties of Cryptographic Hash Functions

Essential Properties

A cryptographic hash function must satisfy four key properties to be considered secure. These properties distinguish cryptographic hashes from simple checksums.

Four Key Properties

Fixed Length Output

The hash always produces the same size output regardless of input size.

Example: SHA-256 always produces 256 bits whether input is 1 byte or 1 GB

One-Way Function

Given a hash value h, it is computationally infeasible to find any input m such that H(m) = h.

Also called: Pre-image Resistance

Avalanche Effect

A small change in input produces a drastically different output.

Flipping one bit in input changes ~50% of output bits

Collision Resistance

It is computationally infeasible to find two different inputs that produce the same hash.

Finding m1 ≠ m2 where H(m1) = H(m2) should be impractical

Common Hash Algorithms Comparison

Algorithm Output Size Block Size Security Status
MD5 128 bits 512 bits Broken (collisions found)
SHA-1 160 bits 512 bits Broken (deprecated)
SHA-256 256 bits 512 bits Secure (widely used)
SHA-3 Variable Variable Secure (newest standard)

MD5 and SHA-1 are Broken

MD5 and SHA-1 have known collision vulnerabilities and should never be used for security purposes. Always use SHA-256 or SHA-3 for cryptographic applications.

4. Non-Cryptographic Hash Functions

What are Non-Cryptographic Hash Functions?

A non-cryptographic hash function is a hash algorithm designed for speed and efficiency, not for security. It converts input data into a fixed-length value mainly for data processing tasks, not for protecting information.

Cryptographic vs Non-Cryptographic Hash Functions

Feature Cryptographic Non-Cryptographic
Purpose Security (integrity, auth) Performance (hash tables, checksums)
Speed Slower (by design) Very fast
Collision Resistance Strong requirement Not required
One-Way Yes (pre-image resistant) No
Examples SHA-256, SHA-3 CRC32, MurmurHash, FNV

Important: Non-cryptographic hash functions should never be used for security purposes. They are designed for speed in data structures (hash tables, bloom filters) and error detection, not for protecting against malicious attacks.

5. Tiger Hash

What is Tiger Hash?

Tiger is a cryptographic hash function designed to be fast in software, particularly on 64-bit processors. It processes input data of any size and produces a fixed-length 192-bit hash value (message digest).

  • Output: 192 bits (24 bytes)
  • Optimized for: 64-bit platforms with high speed
  • Design goals: High speed on 64-bit platforms with strong cryptographic security
  • Use cases: File verification, P2P networks, efficient hashing of large data

Tiger vs SHA-256: Tiger is presented as an example of a modern cryptographic hash function that emphasizes both performance and security. While SHA-256 is the most widely used standard, Tiger shows that hash functions can be designed with specific hardware optimization in mind.

6. HMAC (Hash-Based Message Authentication Code)

What is HMAC?

HMAC is a mechanism that uses a cryptographic hash function together with a secret key to provide message authentication and integrity. Instead of hashing the message alone, HMAC incorporates a secret key, making the hash secure against attackers.

What HMAC Provides

  • Message Integrity: Ensures the message was not altered in transit
  • Sender Authentication: Only someone with the secret key could have generated the correct HMAC
  • Protection Against Tampering: Any modification to the message invalidates the HMAC

HMAC in Practice: HMAC is widely used in security protocols including TLS/SSL, IPSec, API authentication (e.g., AWS request signing), and JWT tokens.

7. Uses of Hash Functions

Real-World Applications

  • Password Storage: Systems store hash values of passwords, not the passwords themselves. When a user logs in, the system hashes the entered password and compares it to the stored hash.
  • Message Integrity (HMAC): Hash functions ensure that a message has not been altered during transmission using HMAC.
  • Digital Signatures: Instead of signing an entire large document, the sender signs the hash of the document. This is much faster and equally secure.
  • Message Fingerprinting: A hash produces a fixed-length "fingerprint" of data. Even a tiny change in the message changes the hash drastically, making tampering detectable.
  • Error Detection: Hash values can reveal accidental changes in data, similar to checksums but cryptographically stronger.
  • Blockchain: Cryptocurrencies like Bitcoin use hash functions (SHA-256) extensively for mining, transaction verification, and linking blocks together.

Hash Functions Are Everywhere: From the HTTPS padlock in your browser to the blockchain behind cryptocurrencies, hash functions are one of the most ubiquitous tools in information security.

Summary and Key Takeaways

What We've Learned

  1. Hash functions convert data of any size into a fixed-length digest. They are one-way functions — you cannot recover the original data from the hash.
  2. Hash functions work by dividing input into blocks, applying padding, processing through a compression function, and producing a fixed-size output.
  3. Cryptographic hash properties include fixed output length, pre-image resistance (one-way), avalanche effect, and collision resistance.
  4. MD5 and SHA-1 are broken and should not be used. SHA-256 and SHA-3 are the current secure standards.
  5. Non-cryptographic hashes (CRC32, MurmurHash) are fast but lack security properties — they are designed for data structures, not protection.
  6. Tiger Hash is a cryptographic hash optimized for 64-bit systems, producing a 192-bit digest.
  7. HMAC combines a secret key with a hash function to provide message authentication and integrity.
  8. Hash functions are used for password storage, digital signatures, message integrity, fingerprinting, error detection, and blockchain technology.
