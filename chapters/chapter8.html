1. Introduction to Authorization

Authorization vs Authentication

Authentication verifies who you are. Authorization determines what you are allowed to do. Authentication must happen first; authorization decides permissions after identity is established.

Key Differences

Aspect Authentication Authorization
Question Who are you? What can you do?
Purpose Verify identity Grant/deny permissions
Order Happens first Happens after authentication
Example Login with username/password User can read files but not delete

Key Insight: Authorization is about controlling access to resources. A properly authenticated user should only have the minimum permissions needed to perform their tasks (the Principle of Least Privilege).

2. Access Control Matrix

What is an Access Control Matrix?

An Access Control Matrix is a conceptual model that defines the permissions of every subject (user/process) for every object (file/resource) in the system.

The matrix has three components:

  • Subjects (rows): Users, processes, or programs requesting access
  • Objects (columns): Files, resources, devices being accessed
  • Actions (cells): Read, Write, Execute, Own, etc.

Access Control Matrix Example

Subject \ Object File A File B Printer
Alice Read, Write, Own Read Print
Bob Read Read, Write, Own Print
Charlie — Read —

Scalability Problem

In practice, the full access control matrix is too large and sparse to store efficiently. With thousands of users and millions of files, most cells would be empty. This leads to two practical implementations: ACLs and C-Lists.

3. Access Control Lists (ACLs)

What is an ACL?

An Access Control List (ACL) stores permissions per object. Each file or resource maintains a list of which subjects can access it and what actions they can perform.

ACLs are the columns of the access control matrix. For each object, we list the subjects and their permissions.

ACL vs Capability List Comparison

ACL (Per Object)

File A:

  • Alice: Read, Write, Own
  • Bob: Read

File B:

  • Bob: Read, Write, Own
  • Charlie: Read

C-List (Per User)

Alice:

  • File A: Read, Write, Own
  • File B: Read
  • Printer: Print

Bob:

  • File A: Read
  • File B: Read, Write, Own

Real-World Example: File system permissions in Linux use ACLs. Each file has owner, group, and other permissions (rwx). Windows NTFS also uses ACLs to control file and folder access.

4. Capability Lists (C-Lists)

What is a Capability List?

A Capability List (C-List) stores permissions per subject. Each user holds a set of tokens (capabilities) that specify which objects they can access and with what permissions.

C-Lists are the rows of the access control matrix. Each subject carries their own list of permissions.

ACL vs C-List Trade-offs

Feature ACL C-List
Organized by Object (resource) Subject (user)
Easy to answer "Who can access this file?" "What can this user access?"
Revocation Easy (edit the file's ACL) Hard (must find all copies of token)
Delegation Harder Easy (pass the token)
Used in File systems (Linux, Windows) Kerberos tickets, OAuth tokens

5. Bell-LaPadula (BLP) Model

What is the BLP Model?

The Bell-LaPadula Model is a formal security model designed to enforce confidentiality in multi-level security systems (e.g., military classified information).

The BLP model assigns security levels to both subjects and objects (e.g., Top Secret > Secret > Confidential > Unclassified) and enforces two key rules:

BLP Model Rules

No Read Up (Simple Security)

🚫📖⬆️

A subject at a lower security level cannot read an object at a higher security level.

Example: A "Secret" user cannot read "Top Secret" documents.

No Write Down (*-Property)

🚫✏️⬇️

A subject at a higher security level cannot write to an object at a lower security level.

Example: A "Top Secret" user cannot write to a "Secret" file (prevents information leaking down).

Purpose: The "No Write Down" rule prevents a high-clearance user (or a Trojan horse running with their privileges) from copying classified information to a lower security level where unauthorized users could read it.

BLP Limitations

The BLP model focuses only on confidentiality, not integrity. It does not prevent a low-level user from writing to a high-level document (which could corrupt it). The Biba model addresses integrity with opposite rules.

6. CAPTCHA

What is CAPTCHA?

CAPTCHA stands for "Completely Automated Public Turing test to tell Computers and Humans Apart." It is a challenge-response test designed to determine whether the user is a human or an automated bot.

How CAPTCHA Works

  • The server generates a challenge that is easy for humans but hard for computers
  • Common types: distorted text, image recognition, audio challenges
  • Modern reCAPTCHA uses behavioral analysis (mouse movements, browsing patterns)

CAPTCHA Applications

  • Account registration: Prevent mass bot account creation
  • Login protection: Block brute-force password attacks
  • Online polls: Prevent vote manipulation
  • Web scraping: Protect content from automated harvesting
  • Ticket purchasing: Prevent scalper bots

CAPTCHA Challenges

CAPTCHAs are an arms race. As AI improves, traditional CAPTCHAs become less effective. Advances in machine learning can now solve many text and image CAPTCHAs. Modern systems rely on behavioral analysis rather than explicit challenges.

7. Firewalls

What is a Firewall?

A firewall is a network security device that monitors and controls incoming and outgoing network traffic based on predetermined security rules. It establishes a barrier between trusted internal networks and untrusted external networks.

Types of Firewalls

Packet-Filtering

Examines each packet's header (source/destination IP, port, protocol) against rules.

  • Fast and simple
  • No session awareness
  • Cannot inspect content

Stateful Inspection

Tracks the state of active connections and makes decisions based on context.

  • Understands connection state
  • More secure than packet-filtering
  • Moderate performance cost

Application Proxy

Acts as an intermediary, inspecting the full application-layer content.

  • Deep content inspection
  • Can filter specific applications
  • Higher latency

Next-Generation (NGFW)

Combines traditional firewall with IDS/IPS, application awareness, and threat intelligence.

  • Deep packet inspection
  • Application identification
  • Integrated threat prevention

Intrusion Detection Systems (IDS)

An IDS monitors network traffic for suspicious activity and generates alerts. Unlike firewalls, IDS is passive — it detects but does not block.

  • Signature-based: Matches known attack patterns
  • Anomaly-based: Detects deviations from normal behavior
  • IPS (Intrusion Prevention System): Active version that can block threats automatically

Personal Firewalls

Software firewalls running on individual computers (e.g., Windows Firewall, macOS Firewall). They protect single devices and can control which applications are allowed to access the network.

Summary and Key Takeaways

What We've Learned

  1. Authorization determines what authenticated users are allowed to do. It is distinct from authentication (identity verification).
  2. The Access Control Matrix is a conceptual model with subjects (rows), objects (columns), and permissions (cells).
  3. ACLs store permissions per object — easy to answer "who can access this resource?"
  4. Capability Lists store permissions per user — easy to answer "what can this user access?" and easy to delegate.
  5. The Bell-LaPadula model enforces confidentiality with two rules: No Read Up and No Write Down.
  6. CAPTCHA distinguishes humans from bots, though it faces increasing challenges from AI advances.
  7. Firewalls range from simple packet-filtering to next-generation systems with deep packet inspection and threat intelligence.
  8. IDS/IPS monitor for intrusions: IDS detects and alerts, IPS actively blocks threats.
