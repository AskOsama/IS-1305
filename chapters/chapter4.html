1. Introduction to Public Key Cryptography

What is Public Key Cryptography?

Public Key Cryptography (PKC), also known as asymmetric cryptography, is a cryptographic system that uses two mathematically related keys: a public key (shared openly) and a private key (kept secret). Data encrypted with one key can only be decrypted with the other.

Public key cryptography solves the fundamental problem of symmetric encryption — key distribution. With symmetric crypto, both parties must somehow share a secret key securely. Public key cryptography eliminates this requirement: anyone can encrypt a message using the recipient's public key, but only the recipient can decrypt it with their private key.

Key Insight: Anyone can encrypt a message using the public key, but only the holder of the private key can decrypt it. This is the foundation of secure communication without prior key exchange.

Symmetric vs Asymmetric Cryptography

Comparison: Symmetric vs Asymmetric

Feature Symmetric Asymmetric
Keys One shared secret key Public + Private key pair
Key Distribution Requires secure channel Public key shared openly
Speed Very fast Much slower
Use Case Bulk data encryption Key exchange, signatures
Examples AES, 3DES RSA, ECC, DH

Best Practice: Public key cryptography enables secure key exchange and authentication, while symmetric cryptography provides efficient data encryption. Modern systems use both together — asymmetric crypto to exchange a session key, then symmetric crypto for fast data encryption.

2. RSA Cryptosystem

What is RSA?

The RSA cryptosystem (named after Rivest, Shamir, and Adleman) is a public key cryptographic system whose security is based on the computational difficulty of factoring large composite integers. It uses a public/private key pair for encryption, decryption, and digital signatures.

RSA Key Generation

  1. Choose two large primes p and q
  2. Compute n = p × q (the modulus)
  3. Compute φ(n) = (p-1)(q-1)
  4. Choose public exponent e such that gcd(e, φ(n)) = 1
  5. Compute private exponent d such that e × d ≡ 1 (mod φ(n))

Public key: (e, n) — shared with everyone

Private key: (d, n) — kept secret

RSA Applications

  • Key Exchange: Securely transmit symmetric session keys
  • Digital Signatures: Sign messages to prove authenticity and non-repudiation
  • Authentication: Verify the identity of communicating parties

Key Size Matters: RSA security depends on key size. A minimum of 2048 bits is recommended today. RSA-1024 is considered insecure, while RSA-4096 provides a higher security margin.

RSA is Slow

RSA encryption is significantly slower than symmetric algorithms like AES. For this reason, RSA is typically not used to encrypt bulk data. Instead, it is used to securely exchange a symmetric key, which is then used for fast data encryption.

3. Diffie-Hellman Key Exchange

What is Diffie-Hellman?

Diffie-Hellman (DH) key exchange is a public-key technique that allows two parties to agree on a shared secret key over an insecure network, without having shared a key beforehand. It does not encrypt data directly — it only establishes a shared secret.

Diffie-Hellman Key Exchange Process

Public Parameters

Both parties agree on: prime p and generator g

Alice

Picks secret a

Computes ga mod p

Sends ga mod p →

Receives gb mod p

Computes (gb)a mod p

Bob

Picks secret b

Computes gb mod p

← Sends gb mod p

Receives ga mod p

Computes (ga)b mod p

Shared Secret

Both arrive at the same value: gab mod p

What DH Provides (and What It Does Not)

Provides

  • Secure key agreement
  • Basis for session keys
  • Foundation for TLS, SSH, IPSec

Does NOT Provide

  • Encryption of data
  • Authentication of parties (by itself)
  • Protection against MITM (alone)

Vulnerable to Man-in-the-Middle

By itself, Diffie-Hellman does not authenticate the communicating parties. An attacker (Trudy) could intercept the exchange and establish separate shared secrets with each party. DH must be combined with authentication (e.g., digital certificates) to be secure.

4. Elliptic Curve Cryptography (ECC)

What is ECC?

Elliptic Curve Cryptography (ECC) is a form of public key cryptography based on the mathematics of elliptic curves over finite fields. ECC provides the same level of security as RSA but with much smaller key sizes, making it faster and more efficient.

Traditional public key systems like RSA require increasingly large key sizes as security needs grow. ECC was developed to improve efficiency, reduce key size, and maintain strong security — especially important for mobile and embedded systems.

RSA vs ECC: Key Size Comparison for Equivalent Security

Security Level (bits) RSA Key Size ECC Key Size RSA / ECC Ratio
80 1024 bits 160 bits 6:1
112 2048 bits 224 bits 9:1
128 3072 bits 256 bits 12:1
256 15360 bits 512 bits 30:1

Where ECC is Used

  • TLS/SSL: Modern HTTPS connections use ECDHE (Elliptic Curve Diffie-Hellman Ephemeral)
  • Mobile and Embedded Systems: Smaller keys mean less processing power and memory
  • Cryptocurrencies: Bitcoin uses the secp256k1 elliptic curve for wallet keys
  • Secure Messaging: Signal Protocol uses Curve25519 for key exchange

Why ECC Matters: A 256-bit ECC key provides comparable security to a 3072-bit RSA key. This makes ECC the preferred choice for resource-constrained environments and modern protocols where efficiency is critical.

5. Public Key Notation

Standard Notation for Public Key Operations

Public key cryptography uses standardized notation to distinguish between encryption, decryption, and signing operations. The notation identifies which key is used and who owns it.

Public Key Notation Guide

{ } Curly Braces — Public Key

{M}Alice

Encrypt message M with Alice's public key

Anyone can do this

[ ] Square Braces — Private Key

[M]Alice

Encrypt/sign message M with Alice's private key

Only Alice can do this

Notation Examples

  • Encryption: C = {M}Alice — Encrypt message M with Alice's public key. Only Alice can decrypt.
  • Decryption: M = [C]Alice — Decrypt ciphertext C with Alice's private key.
  • Digital Signature: S = [M]Alice — Alice signs message M using her private key. Anyone can verify using Alice's public key.

Digital Signatures and Non-Repudiation: When Alice signs a message with her private key [M]Alice, anyone can verify the signature using her public key. Since only Alice has her private key, she cannot deny having signed the message — this is called non-repudiation.

6. Public Key Infrastructure (PKI)

What is PKI?

Public Key Infrastructure (PKI) is the framework that manages public keys and digital certificates to enable secure communication using public key cryptography. PKI provides a trusted way to verify that a public key genuinely belongs to its claimed owner.

Why PKI is Needed

Without PKI, public keys are vulnerable to Man-in-the-Middle (MITM) attacks:

  1. Fahad shares his public key with Fatima
  2. Attacker Sara intercepts and replaces it with her own public key
  3. Fatima encrypts messages thinking she's using Fahad's key — but Sara can read them
  4. Sara re-encrypts with Fahad's real key and forwards, undetected

PKI prevents this by having a trusted third party verify and certify public key ownership.

Main Components of PKI

  • Certificate Authority (CA): Trusted entity that issues digital certificates. Verifies identity before binding a public key to a user.
  • Registration Authority (RA): Assists the CA by verifying user identity before certificate issuance.
  • Digital Certificates: Electronic documents that link a public key with an identity (e.g., SSL/TLS website certificates, X.509 format).
  • Certificate Repository: Database where certificates are stored and accessed by relying parties.

Trust Model: Just as a government agency (like DMV) verifies identity and issues an ID card, the RA confirms identity and the CA issues a digitally signed certificate. Even if parties don't know each other, they can trust the CA that certified them.

Summary and Key Takeaways

What We've Learned

  1. Public key cryptography uses two mathematically related keys (public and private) and solves the key distribution problem of symmetric cryptography.
  2. RSA is the most widely known public key algorithm, based on the difficulty of factoring large primes. It is used for key exchange, digital signatures, and authentication.
  3. Diffie-Hellman allows two parties to establish a shared secret over an insecure channel, but does not provide authentication by itself.
  4. Elliptic Curve Cryptography (ECC) provides the same security as RSA with much smaller keys, making it ideal for modern and resource-constrained systems.
  5. Public key notation uses curly braces {M} for public key operations and square brackets [M] for private key operations.
  6. Digital signatures provide non-repudiation — the signer cannot deny having signed the message.
  7. PKI (Certificate Authorities, Registration Authorities, and digital certificates) provides the trust framework for verifying public key ownership and preventing MITM attacks.
