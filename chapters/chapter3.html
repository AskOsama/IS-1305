1. Introduction to Symmetric Key Cryptography

What is Symmetric Key Cryptography?

Symmetric key cryptography is the oldest and most widely used form of encryption. Both the sender and the receiver share the same secret key for both encryption and decryption.

Symmetric key cryptography is the workhorse of modern security systems. It is fast, efficient, and used to encrypt the vast majority of data in transit and at rest.

Key Challenge: The fundamental problem with symmetric key cryptography is key distribution — how do you securely share the secret key with the other party? If an attacker intercepts the key, all communications are compromised.

Two Families of Symmetric Ciphers

Symmetric key algorithms are divided into two main categories:

Stream Ciphers vs Block Ciphers

Stream Ciphers

Encrypt one bit (or byte) at a time

  • Generate a keystream from the key
  • XOR keystream with plaintext
  • Fast, low latency
  • Examples: RC4, A5/1

Block Ciphers

Encrypt a fixed-size block at a time

  • Process data in fixed blocks (64 or 128 bits)
  • Multiple rounds of substitution and permutation
  • More versatile, widely standardized
  • Examples: DES, 3DES, AES

2. Stream Ciphers

How Stream Ciphers Work

A stream cipher generates a pseudo-random keystream from the secret key, then combines it with the plaintext one bit or byte at a time using XOR. The same keystream is used to decrypt.

RC4 (Rivest Cipher 4)

RC4 is a widely deployed stream cipher, once used in:

  • SSL/TLS: Securing web traffic (now deprecated)
  • WEP: Wi-Fi encryption (completely broken)
  • WPA-TKIP: Temporary Wi-Fi fix (also deprecated)

RC4 uses a variable-length key (40–2048 bits) and generates one byte of keystream per operation. It is fast and simple to implement.

RC4 is Broken

RC4 has known biases in its keystream output and should no longer be used. It was officially prohibited in TLS by RFC 7465 (2015). All modern systems have moved to AES-based ciphers.

A5/1 (GSM Encryption)

A5/1 is a stream cipher used to encrypt voice communications in the GSM (2G) mobile phone standard.

  • Uses three Linear Feedback Shift Registers (LFSRs)
  • 64-bit key, generates 1 bit of keystream per clock cycle
  • Designed for hardware efficiency in mobile phones

A5/1 Vulnerabilities

A5/1 has been completely broken. Known attacks include time-memory trade-off attacks that can crack GSM calls in near real-time. Modern mobile networks (4G/5G) use much stronger encryption.

One-Time Pad (OTP)

The One-Time Pad is the only theoretically unbreakable cipher. It uses a random key that is:

  • At least as long as the plaintext message
  • Truly random (not pseudo-random)
  • Used only once and then destroyed
  • Known only to sender and receiver

Why OTP Is Impractical: While theoretically perfect, the One-Time Pad requires a key as long as the message itself, making key distribution extremely difficult. If you could securely deliver a key that long, you could just deliver the message itself!

3. Block Ciphers

How Block Ciphers Work

A block cipher encrypts data in fixed-size blocks. Each block goes through multiple rounds of substitution (confusion) and permutation (diffusion) operations controlled by the key.

DES (Data Encryption Standard)

DES was the first commercially adopted encryption standard, published by NIST in 1977.

  • Block size: 64 bits
  • Key size: 56 bits (64-bit key with 8 parity bits)
  • Rounds: 16 rounds of Feistel network operations
  • Structure: Each round uses substitution (S-boxes) and permutation (P-boxes)

DES is Insecure

With only a 56-bit key, DES can be brute-forced. In 1999, a DES key was cracked in just 22 hours using specialized hardware. DES should never be used for new systems.

3DES (Triple DES)

3DES applies DES three times with two or three different keys to increase security.

  • Process: Encrypt → Decrypt → Encrypt (EDE mode)
  • Key size: 112 bits (2-key) or 168 bits (3-key)
  • Block size: 64 bits (unchanged from DES)
  • Speed: Three times slower than DES

Why EDE? The Encrypt-Decrypt-Encrypt pattern was chosen so that 3DES is backward compatible with single DES when all three keys are the same (the middle decrypt undoes the first encrypt).

AES (Advanced Encryption Standard)

AES replaced DES as the standard symmetric cipher. Selected by NIST in 2001 from the Rijndael algorithm, it is the most widely used cipher today.

  • Block size: 128 bits
  • Key sizes: 128, 192, or 256 bits
  • Rounds: 10 (128-bit key), 12 (192-bit), or 14 (256-bit)
  • Structure: Substitution-permutation network (not Feistel)

AES is the Standard: AES is used everywhere — TLS/HTTPS, Wi-Fi (WPA2/WPA3), disk encryption (BitLocker, FileVault), VPNs (IPSec), and more. AES-256 is approved for Top Secret classified data by the U.S. government.

4. Integrity and MACs

What is a MAC?

A Message Authentication Code (MAC) is a short piece of information used to verify both the integrity and the authenticity of a message. It is computed using a secret key shared between sender and receiver.

While encryption provides confidentiality, it does not guarantee that data has not been tampered with. MACs address this gap.

Common MAC Algorithms

  • HMAC: Hash-based MAC using algorithms like SHA-256. The most widely used MAC construction.
  • CMAC: Cipher-based MAC built on block ciphers like AES.
  • GMAC: Galois MAC, used in AES-GCM authenticated encryption.

Best Practice: Modern systems use authenticated encryption (e.g. AES-GCM) which combines encryption and MAC in a single operation, providing both confidentiality and integrity efficiently.

Summary and Key Takeaways

What We've Learned

  1. Symmetric key cryptography uses the same shared secret key for encryption and decryption. It is the fastest form of encryption.
  2. Stream ciphers (RC4, A5/1) encrypt data one bit/byte at a time using a keystream XORed with plaintext.
  3. The One-Time Pad is theoretically unbreakable but impractical because the key must be as long as the message.
  4. Block ciphers (DES, 3DES, AES) encrypt fixed-size blocks through multiple rounds of substitution and permutation.
  5. DES (56-bit key) is broken; 3DES (112/168-bit key) is deprecated; AES (128/192/256-bit key) is the current standard.
  6. MACs provide data integrity and authentication. HMAC (using hash functions) and CMAC (using block ciphers) are common constructions.
  7. Authenticated encryption (AES-GCM) combines confidentiality and integrity in one efficient operation.
